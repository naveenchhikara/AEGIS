# Plan 04-02: Translation Content for All UI Labels

**Wave:** 2 of 7
**Autonomous:** true
**Depends on:** 04-01 (i18n foundation)
**Blocked by:** None

## Overview

Generate comprehensive translations for all UI labels across English, Hindi, Marathi, and Gujarati. This wave fills the translation files with content covering navigation, status badges, table headers, buttons, and common UI elements.

## Tasks

### Task 1: Complete English translation base
1. Fill `src/i18n/locales/en/common.json` with all labels:
   - **Navigation**: dashboard, compliance, auditPlan, findings, boardReport, auditeePortal, settings
   - **Compliance**: requirement, category, status, deadline, assignee, evidence, compliant, partial, nonCompliant, pending
   - **Audit**: auditName, branch, department, plannedDates, team, progress, notStarted, inProgress, onHold, complete
   - **Findings**: findingId, title, severity, critical, high, medium, low, observation, rootCause, riskImpact, auditeeResponse, actionPlan
   - **Report**: executiveSummary, auditCoverage, keyFindings, complianceScorecard, recommendations
   - **Status**: draft, underReview, responsePending, actionPlanned, closed
   - **Common**: loading, error, save, cancel, delete, export, print, search, filter, sortBy
   - **Login**: email, password, mfaCode, loginButton, language
   - **Dashboard**: healthScore, auditCoverage, totalFindings, criticalFindings, openFindings, overdueFindings

### Task 2: Generate Hindi translations
1. Fill `src/i18n/locales/hi/common.json` with Hindi translations:
   - Use Devanagari script throughout
   - Technical terms remain in English where appropriate (RBI, Audit, CEO)
   - Example: "Compliance Registry" → "अनुपालन रजिस्टर"
   - Example: "Dashboard" → "डैशबोर्ड"
   - Example: "Findings" → "निष्कर्ष"
   - Example: "Status" → "स्थिति"
2. Preserve JSON structure exactly matching English

### Task 3: Generate Marathi translations
1. Fill `src/i18n/locales/mr/common.json` with Marathi translations:
   - Use Devanagari script (Marathi variant)
   - Distinguish from Hindi where terminology differs
   - Example: "Compliance Registry" → "अनुपालन नोंदणी"
   - Example: "Findings" → "आढळलेली तक्रार"
   - Example: "Audit Plan" → "लेखापरीक्षा योजना"
2. Preserve JSON structure exactly matching English

### Task 4: Generate Gujarati translations
1. Fill `src/i18n/locales/gu/common.json` with Gujarati translations:
   - Use Gujarati script
   - Example: "Compliance Registry" → "અનુપાલન નોંધણી"
   - Example: "Findings" → "શોધ"
   - Example: "Audit Plan" → "ઓડિટ યોજના"
   - Example: "Dashboard" → "ડેશબોર્ડ"
2. Preserve JSON structure exactly matching English

### Task 5: Create translation validation script
1. Create `src/i18n/validate.ts` that:
   - Imports all 4 locale files
   - Checks that all keys exist in all locales
   - Reports missing translations
   - Runs as part of build or via `npm run validate:i18n`
2. Add script to `package.json`: `"validate:i18n": "tsx src/i18n/validate.ts"`
3. Run validation: All keys present across all 4 languages

### Task 6: Add translation coverage note
1. Create `src/i18n/README.md` documenting:
   - Which keys use AI-generated translations
   - Which keys need professional review (marked with TODO comments)
   - Banking terminology requiring domain expert verification
2. Mark compliance-related translations as needing expert review

## Must Haves

- All 4 translation files contain complete translations for every key
- JSON structure is identical across all locales (no missing keys)
- Validation script runs without errors
- README documents translation quality status
